# ByteBuddy - Quick Setup Guide

## 5-Minute Setup

### Prerequisites

- Java 17+
- Node.js 18+
- MySQL 8+ (or Docker)

### Step 1: Get API Key

1. Visit [console.groq.com](https://console.groq.com/keys)
2. Sign up (free)
3. Copy your API key

### Step 2: Configure Backend

Edit `backend/src/main/resources/application.properties`:

```properties
groq.api.key=YOUR_API_KEY_HERE
```

### Step 3: Setup Database

**Option A: Using Docker (recommended)**

```bash
docker run -d \
  --name mysql-chatbot \
  -e MYSQL_ROOT_PASSWORD=Raghukanch007@ \
  -e MYSQL_DATABASE=chatbot_db \
  -p 3306:3306 \
  mysql:8.0
```

**Option B: Using MySQL CLI**

```bash
mysql -u root -p
CREATE DATABASE chatbot_db;
```

### Step 4: Run Backend

```bash
cd backend
gradle bootRun
```

Wait for: `Started ChatGptCloneApplication in X seconds`

### Step 5: Run Frontend

```bash
cd frontend
npm install
npm start
```

Browser opens to `http://localhost:3000` ‚úì

## Key Features

| Feature      | Details                            |
| ------------ | ---------------------------------- |
| **Chat**     | Send messages, get AI responses    |
| **History**  | All messages saved in database     |
| **Sessions** | Separate conversations with titles |
| **Sidebar**  | Manage & rename conversations      |
| **Context**  | AI remembers entire conversation   |

## API Quick Reference

### Send Message

```bash
curl -X POST http://localhost:8081/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role": "user", "content": "Hello"}],
    "sessionId": "session-123"
  }'
```

### Get Conversation

```bash
curl http://localhost:8081/api/conversation/session-123
```

### List All Conversations

```bash
curl http://localhost:8081/api/conversations
```

## Troubleshooting

| Problem                | Solution                                        |
| ---------------------- | ----------------------------------------------- |
| Backend won't start    | Check Java 17+: `java -version`                 |
| Frontend won't load    | Ensure backend running: `gradle bootRun`        |
| No database connection | Create database: `mysql -u root -p < setup.sql` |
| Groq API error         | Check API key in `application.properties`       |

## Project Files

**Frontend:**

- `frontend/src/App.js` - Main chat component
- `frontend/package.json` - Dependencies
- `frontend/public/` - Static files

**Backend:**

- `backend/src/main/java/.../ChatController.java` - API endpoints
- `backend/src/main/java/.../ChatService.java` - Business logic
- `backend/src/main/resources/application.properties` - Configuration

**Database:**

- SQL auto-generated by Spring Boot
- Tables: `conversations`, `chat_messages`
